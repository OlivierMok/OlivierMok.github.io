Code Log 4: CSS Color

What do these CSS properties do? (color, background-color, border-color)

The CSS color property sets the foreground color of an element's text content and text decorations, such as underlines or overlines, making it essential for readability and styling text. The background-color property defines the background color behind an element's content and padding (but not margins), which helps create contrast, highlight sections, or add visual depth to layouts. Finally, border-color specifies the color of an element's border, allowing designers to outline boxes, tables, or other elements for separation, emphasis, or decorative purposes; it can be set for all sides uniformly or individually (e.g., border-top-color).

How does CSS accept color as a value? (Examples in named, hexadecimal, RGB, and other formats)

CSS accepts colors in several formats, allowing flexibility for precision, simplicity, or compatibility. The most common include named colors for basic use, hexadecimal for web-safe shorthand, RGB/RGBA for fine-tuned values with optional transparency, and others like HSL/HSLA for intuitive hue-based adjustments. Here are examples:

  • Named: Uses predefined keywords like red, blue, or transparent (e.g., color: red; for basic red text).
  • Hexadecimal: A 3- or 6-digit code starting with #, representing red/green/blue channels (e.g., background-color: #ff0000; for pure red, or #f00 shorthand).
  • RGB: Specifies red, green, blue values from 0-255, with optional alpha for transparency in RGBA (e.g., border-color: rgb(255, 0, 0); for red, or rgba(255, 0, 0, 0.5); for semi-transparent red).
  • Other formats (e.g., HSL/HSLA): Hue (0-360°), saturation (0-100%), lightness (0-100%), with alpha in HSLA (e.g., color: hsl(0, 100%, 50%); for red, or hsla(0, 100%, 50%, 0.7); for translucent red).
