Code Log 1: HTML Semantic Tags

What is the difference between non-semantic and semantic tags in HTML?

Non-semantic tags are generic containers that carry no meaning about the content they hold, they exist mainly for styling or grouping.
Semantic tags describe the purpose and role of the content, giving it clear meaning to browsers, search engines, and screen readers.
The key difference is that semantic tags communicate what the content is (e.g., navigation, main content, sidebar), while non-semantic tags only say how it should be grouped or styled.

What are two (or more) benefits of using semantic tags?

  • They improve accessibility by helping screen readers understand and navigate the page structure more effectively.
  • They boost SEO because search engines can better interpret the content hierarchy and context of the page.
  • They make code more readable and maintainable since the tag names instantly show the purpose of each section.
  • They provide better future-proofing and consistency across browsers and devices by following modern web standards.
  • They enable more meaningful default behaviors in browsers and assistive tools (e.g., jumping to main content or landmark regions).
