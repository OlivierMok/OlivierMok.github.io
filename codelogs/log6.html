Code Log 6: Code References

What did you look up?

I researched the CSS Flexbox layout model, specifically the display: flex property and its core concepts (flex container, flex items, main axis, cross axis, and common properties like justify-content, align-items, and flex).

What sites did you consult?

1. W3Schools
https://www.w3schools.com/css/css3_flexbox.asp

2. MDN Web Docs (Mozilla)
https://developer.mozilla.org/en-US/docs/Learn/CSS/CSS_layout/Flexbox

3. CSS-Tricks
https://css-tricks.com/snippets/css/a-guide-to-flexbox/

4. web.dev (Google)
https://web.dev/learn/css/flexbox

What are the differences in the information presented on each site?

All four sites explain the same core concepts, but they differ in depth, presentation style, and focus:

  • W3Schools is the most beginner-friendly. It uses very simple language, lots of “Try it Yourself” live editors, and short code snippets. It focuses on quick examples but has the least explanation of why things work.
  • MDN is the most comprehensive and technically accurate. It includes detailed terminology, browser compatibility tables, links to the official CSS specification, and thorough examples. It feels like a formal reference manual.
  • CSS-Tricks stands out for its visual, practical approach. It includes many real-world layout patterns, colorful diagrams, and a famous “Flexbox Cheatsheet” style that makes it easy to visualize properties.
  • web.dev is modern and performance-oriented. It emphasizes how Flexbox affects layout performance and includes interactive demos and best-practice tips for responsive design.

What features do you think a code reference page should offer?

A good code reference page should include:

  • Clear, concise explanations with visual diagrams
  • Live, editable code examples (like W3Schools and CSS-Tricks)
  • Browser compatibility information
  • Common use cases and real-world examples
  • Search functionality and quick navigation
  • Links to the official specification and related properties
  • Accessibility and performance notes

Do any of your 4 resources stand out as particularly good or bad?

MDN is the best overall — it is the most authoritative, up-to-date, and complete reference. It’s the one I trust most for accurate technical details. CSS-Tricks is my favorite for learning because of its excellent visuals and practical examples. W3Schools is great for absolute beginners who want to experiment quickly, but it sometimes lacks depth. web.dev is excellent for modern best practices but is slightly less comprehensive than MDN.

None of them are truly “bad,” but W3Schools feels the most limited for advanced users.
