Code Log 5: Images

1. What are the differences between presenting an image using an HTML tag (like <img>) and presenting it as the background of an HTML element (using the CSS property background-image)?

In HTML, images are typically presented using the <img> tag, which embeds the image directly as inline or block content within the document flow. The code looks like <img src="image-url" alt="description">, and it allows for attributes like width, height, and alt for accessibility. Functionally, <img> is semantic, meaning it's treated as content (e.g., screen readers announce it), and it can be resized or styled but always occupies space in the layout. For example:

In contrast, presenting an image as a CSS background-image applies it to the background of an element (e.g., div) via background-image: url('image-url');. This is non-semantic—it's decorative, not content—so screen readers ignore it, and it's ideal for overlays or repeated patterns. 2. Besides background-image, what other CSS properties affect the display of background images? Describe at least two more CSS properties.

Besides background-image, several CSS properties control how background images are displayed. Two key ones are:

  • background-size: This sets the size of the background image relative to its container. Values like cover scale the image to fill the element while maintaining aspect ratio (cropping if needed), contain fits the entire image inside without cropping, or specific dimensions like 100px 200px. It's useful for responsive designs.
  • background-position: This positions the background image within its element. Values like center, top left, or percentages (e.g., 50% 0%) control alignment, making it ideal for focal points or parallax effects.

3. In one sentence, how would you define the differences between dimensions and resolution in a bitmap image?

Dimensions refer to the width and height of an image in pixels (e.g., 1920x1080), defining its size on screen, while resolution is the density of pixels per inch (PPI/DPI), determining sharpness and quality when printed or viewed at actual size.

4. Extra Credit: Similarities and differences between image file formats (GIF, PNG, JPG/JPEG, WebP, SVG)

All these formats store visual data for web use, but they differ in compression, transparency, animation, and scalability. Similarities include wide browser support and raster/vector capabilities for images.

  • GIF: Lossless compression, supports animation and transparency (but only 256 colors); best for simple graphics/animations like memes, but large files and limited colors make it poor for photos.
  • PNG: Lossless, supports transparency and high colors; ideal for logos/icons with sharp edges, but files are larger than lossy formats, making it suboptimal for photos.
  • JPG/JPEG: Lossy compression, no transparency; excellent for photos with small file sizes and good quality, but artifacts appear on repeated saves, unsuitable for text/graphics.
  • WebP: Lossy/lossless, supports transparency/animation; combines JPEG quality with PNG features at smaller sizes, great for web performance, but older browsers may need fallbacks.
  • SVG: Vector-based (XML), scalable without quality loss, supports animation/transparency; perfect for logos/icons that resize, but not for complex photos (rasterize to bitmap for those).
