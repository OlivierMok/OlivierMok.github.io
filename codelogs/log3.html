Code Log 3: CSS Media Queries

1. How does a CSS media query work, and why is it useful?

A CSS media query is a conditional rule that applies different styles based on the characteristics of the user's device or browser environment. It uses the @media rule, followed by a media type (e.g., screen, print) and/or media features (conditions like screen width), and then a block of CSS rules that only activate when the condition is true. For example: @media (max-width: 768px) { body { font-size: 16px; } }. This means the styles inside only apply when the viewport is 768px wide or narrower. Media queries are essential for responsive web design â€” they allow a single website to adapt its layout, typography, and behavior to phones, tablets, laptops, and desktops without needing separate mobile versions. This improves usability, accessibility, and performance across all devices.

2. What "media features" are supported by media queries? Pick any 4 to explain.

CSS media queries support many media features that describe the device or viewport. Here are four commonly used ones:

  • max-width / min-width: Checks the viewport width. max-width: 768px targets screens narrower than or equal to 768px (typical for tablets and phones); min-width: 1024px targets larger screens (desktops).
  • orientation: Detects whether the device is in portrait (orientation: portrait) or landscape (orientation: landscape) mode. Useful for adjusting layouts when a phone is rotated.
  • prefers-color-scheme: Detects the user's system preference for light or dark mode. prefers-color-scheme: dark lets you apply dark theme styles automatically if the user has enabled dark mode on their device.
  • hover: Checks if the primary input device can hover (e.g., mouse) or not (e.g., touch). hover: hover applies hover effects only on devices with a mouse; hover: none can hide or replace hover states on touch devices.
